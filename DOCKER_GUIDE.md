🐳 Docker 실행 및 프로젝트 구조 가이드
이 문서는 도커를 이용해 AI 서버 환경을 구축하고 실행하는 방법을 안내합니다. 모든 팀원은 동일한 도커 환경에서 작업하여 라이브러리 충돌을 방지합니다.

1. 🏗️ 프로젝트 폴더 구조 설명
우리 프로젝트의 뼈대는 다음과 같이 구성되어 있습니다. 각 폴더의 역할을 확인하고 정해진 위치에 코드를 작성해 주세요.

```text
EMERGENCY_VOICE_RESCUE
├── src/                # 모든 소스 코드가 들어가는 곳
│   ├── audio/          # 실시간 음성 탐지 (TinyML) 로직
│   ├── ml/             # 오디오 분류 모델 관련
│   ├── nlp/            # Gemma-3 (LLM) 및 의료 Q&A 처리
│   ├── voice/          # TTS 안내 방송 관련
│   └── main.py         # 전체 시스템을 실행하는 통합 컨트롤러
├── data/               # 데이터셋 (Git 제외 대상)
│   ├── train_set/      # 학습용 원본 오디오
│   └── preprocessed/   # 전처리된 .npy 파일들
├── models/             # 학습된 모델 가중치 파일
├── Dockerfile          # AI 환경 설정 설계도
├── docker-compose.yaml # 멀티 컨테이너 실행 설정
└── requirements.txt    # 필요한 파이썬 라이브러리 목록
```

2. 🚀 도커 실행 방법
노트북에 Docker Desktop이 설치되어 있어야 하며, NVIDIA GPU를 사용한다면 최신 드라이버가 깔려 있어야 합니다.

1단계: 이미지 빌드 및 컨테이너 실행
터미널에서 프로젝트 루트 폴더로 이동한 후 아래 명령어를 입력하세요.

Bash
# 처음 실행하거나 requirements.txt가 수정되었을 때
docker-compose up --build -d
-d: 백그라운드에서 실행합니다. (터미널을 계속 사용할 수 있음)

--build: 환경 설정(Dockerfile)이 바뀌었을 때 강제로 새로 만듭니다.

2단계: 실행 중인 컨테이너 접속 (터미널 작업)
코드 수정을 완료하고 직접 실행해 보려면 컨테이너 내부로 들어가야 합니다.

Bash
# 컨테이너 내부 터미널 접속
docker exec -it rescue_system_container /bin/bash
3단계: 파이썬 코드 실행 (컨테이너 안에서)
컨테이너에 접속했다면 평소처럼 파이썬 명령어를 사용할 수 있습니다.

Bash
# 메인 시스템 실행
python src/main.py

# 특정 모듈 테스트 (예: 음성 전처리)
python src/audio/preprocess_audio.py
3. ⚠️ 팀원 주의사항 (중요)
💾 데이터 파일 위치
data/ 폴더 안의 파일들은 깃허브에 올라가지 않습니다.

팀원들이 전처리한 .npy 파일이나 학습 데이터를 받았다면, 본인 노트북의 data/ 폴더 안에 직접 넣어줘야 도커 내부에서 인식합니다.

📦 라이브러리 추가
새로운 라이브러리(import가 필요한 것들)를 추가했다면, 반드시 로컬의 requirements.txt에 적어주세요.

다른 팀원들은 docker-compose up --build를 통해 해당 라이브러리를 자동으로 설치하게 됩니다.

🔋 노트북 성능 관리
Gemma-3 모델은 사양을 많이 탑니다. 실행 중 노트북이 너무 뜨거워지면 전원을 연결하고 통풍이 잘되는 곳에서 작업하세요.
