핵심 기능
조난 상황이 발생하면 구조 요청자는 “살려주세요”, “여기 있어요” 등의 음성을 통해 구조 요청을 한다. 시스템은 해당 음성을 인식하여 구조 요청 여부를 판단하고 자동으로 활성화된다. 이를 통해 설치된 기기가 조난자의 음성 구조 요청을 감지하면, 저전력 통신망(LPWA)을 통해 산 아래 관제센터나 위성 단말기로 "기기의 좌표"를 우선적으로 119 및 산악 구조대에 송신한다.
현재 조난자의 상태를 묻는 질문을 통해 현재 상태를 음성으로 인식받으면 119 및 산악 구조대에 2차 송신한다.
이 과정에서 음성 데이터는 텍스트 형태로 변환되어 전달되며, 사용자의 위치와 상태가 함께 포함된다. 이후 추가로 입력되는 음성 역시 지속적으로 분석되어 새로운 정보가 갱신되고, 해당 내용은 구조 기관에 반복적으로 전달된다.
또한 시스템은 사용자의 음성을 기반으로 상태를 판단하고, 필요한 경우 간단한 응급처치 방법을 음성으로 안내한다. 이를 통해 구조 요청자는 구조가 도착하기 전까지 기본적인 대응을 할 수 있다. 전체 과정은 구조 완료 시까지 반복되며, 구조자는 실시간으로 위치와 상태 정보를 전달받아 보다 신속하게 구조를 진행할 수 있다.

상황예시:
구조자가 살려주세요! 등 구조를 음성으로 외침 -> 음성인식을 통해 gps랑 구조메세지 산악구조대, 119 전송
구조자 상태 파악을 위한 안내멘트 -> 구조자가 어디어디 아프다 음성으로 전달 -> 산악구조대, 119 전송 및 ai가 판단해서 응급처지 안내(혹은 구조대에서 보내는 응급처지내용(텍스트던 뭐던)을 음성으로 재 안내)
응급구조대(산악구조대, 119)에서 현 상황(구조하러가는 실시간 상황) 전송 -> 구조대기 중인 구조자한테 안내멘트
구조자가 기다리면서 느낄 불안감 해소를 위한 방안

테스트 구현 방법 : 

구조자 음성 구조 요청 → [음성인식장치] → 음성 감지 → 로컬 처리 (VAD, 키워드 감지) → STT 변환 → 핵심 정보 추출 → 데이터 전송(좌표) → [중계 서버 / 메시 네트워크] → 데이터 수신 및 저장 → 구조대 전달 → [구조대 시스템] → 대시보드 확인 → 출동
구조자 상태 파악 안내 멘트(기기 송출) → 조난자 상태 음성 답변 (예: "다리가 부러졌어") → [음성인식장치] → 음성 입력 → 로컬 처리 → STT 변환 및 의도 분석 → 부상 부위 및 심각도 추출 → 데이터 전송 → [중계 서버] → 구조대에 부상 정보 업데이트 → [기기 자체 처리] → AI 응급처치 TTS 안내 (예: "지혈을 위해 상처 윗부분을 묶으세요")
[구조대 시스템] → 구조대원이 실시간 이동 상황 입력 (예: "5분 뒤 도착 예정") 또는 음성 입력 → 텍스트 데이터 변환 → 데이터 전송 (구조대 → 기기) → [음성인식장치(비콘)] → 데이터 수신 → TTS 음성 변환 (안정감을 주는 차분한 목소리) → 구조자에게 안내 멘트 송출 (예: "구조대가 현재 500m 앞까지 왔습니다. 조금만 더 힘내세요.")

역할분담 : 
입력 (음성/디바이스) - 김서진
AI 처리 (STT/NLP/판단) - 송태능
통신 (저전력/메시지) - 이호숙
서버/백엔드 (데이터 흐름) - 채형찬
UI/대시보드 (구조대 + 사용자) - 최서언
