# 음성 기반 산악 구조 시스템

## 프로젝트 개요
- **프로젝트명**: 음성 기반 산악 구조 시스템 
- **팀명**: 6팀  

---

## 기획 배경
산악 조난 상황에서는  
- 통신 불안정  
- 부상으로 인한 신체 제약  
- 휴대폰 사용 불가  

등의 문제로 구조 요청이 지연되며, 이는 **생존율 감소**로 이어짐.

---

## 문제 정의
- 구조 요청 자체가 어려운 환경
- 위치 및 상태 전달 지연
- 실시간 대응 어려움

---

## 타겟 사용자
- 등산객 (조난자)
- 119 및 산악 구조대

---

## 사용 환경
- 산악 지역
- 조난, 실족, 고립 상황

---

## 핵심 기능 요약

### 1. 음성 기반 구조 요청
- “살려주세요”, “여기 있어요” 음성 감지
- 자동 구조 요청 활성화

### 2. 위치 및 데이터 전송
- GPS 좌표 우선 전송
- LPWA / 위성 통신 활용

### 3. 상태 파악 및 전달
- 음성 응답 → 텍스트 변환(STT)
- 부상 상태 및 상황 정보 전송

### 4. 응급처치 안내
- AI 또는 구조대 메시지를 TTS로 전달

### 5. 구조 진행 안내
- 구조대 이동 상황 음성 안내
- 심리적 안정 제공

---

## 시스템 흐름

1. 음성 감지 → STT 변환  
2. 핵심 정보 추출 → 좌표 전송  
3. 서버 전달 → 구조대 확인  
4. 구조대 응답 → TTS 안내  
5. 반복 업데이트  

---

## 사용 모델

| 구분 | 모델 | 역할 |
|------|------|------|
| 음성 감지 | TinyML | 저전력 음성 트리거 |
| STT | Vosk / Faster Whisper | 음성 → 텍스트 |
| 정보 추출 | DistilKoBERT | 핵심 정보 추출 |
| 상황 판단 | Llama / Gemma | 요약 및 보고 |
| 긴급도 분류 | ML 모델 | 우선순위 판단 |
| TTS | Piper | 음성 안내 |

---

## 모델 선정 이유
- 오프라인 동작 가능
- 저전력 환경 적합
- 한국어 처리 가능
- 네트워크 의존 최소화

---

## 기대 효과
- 구조 요청 시간 단축
- 위치 및 상태 정확 전달
- 구조 속도 향상
- 조난자 불안 감소

---

## 역할 분담
- 입력(디바이스): 김서진  
- AI 처리: 송태능  
- 통신: 이호숙  
- 서버/백엔드: 채형찬  
- UI/대시보드: 최서언
